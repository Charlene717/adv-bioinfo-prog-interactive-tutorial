可再現研究：讓 5 年後的你還跑得起來

「我跑得起來但別人跑不起來」是學界長期之痛——本章涵蓋 5 級再現性階梯、Quarto / Jupyter、環境鎖檔、FAIR data，以及讓論文真正能被別人 (與未來的你) 重現的工程實踐。

"It works on my machine but not theirs" is a chronic problem in academia. This chapter covers the 5-level reproducibility ladder, Quarto/Jupyter, env lockfiles, FAIR data, and engineering practices that make papers actually reproducible — by others and by future-you.

概念

再現性的五個階梯

學界對「reproducible」一詞長期混淆。Stodden、Peng 等人的分類提供了清晰的階梯——本章建議以此為目標分級：

Academia has long conflated "reproducible." A widely-cited taxonomy by Stodden, Peng et al. gives a clean ladder — use these tiers as goals:

Re-run — 你自己能再跑一次

最低門檻。注意：很多 Jupyter notebook 在自己機器上時隔 6 個月後也跑不起來——因為套件升級。

The lowest bar. Note: many Jupyter notebooks can't be rerun even by their author 6 months later — package upgrades break them.

Re-execute — 別人能在同樣資料跑出同樣結果

需要：完整 code、原始資料、環境鎖檔。論文 supplementary 應該至少達到這個等級。

Requires: complete code, raw data, environment lockfile. Paper supplementaries should reach at least this tier.

Reproduce — 別人從相同方法做出相同結論

允許不同實作 (例如改用 DESeq2 換成 edgeR) 但仍得到一致結論。考驗的是方法描述清晰度。

Allows different implementations (e.g., DESeq2 swapped for edgeR) but same conclusions. Tests methods clarity.

Replicate — 別人用新資料得到一致結論

需要新獨立資料 (independent cohort)。是生物學發現的「黃金標準」。

Requires an independent dataset. The gold standard of biological findings.

Generalize — 結論在更廣的範圍仍成立

跨物種、跨組織、跨疾病的廣泛驗證。論文的最終價值在此。

Validation across species, tissues, diseases. Where a paper's lasting value lives.

📘
本章重點：第 1-3 級是「程式工程問題」，是本章可解決的。第 4-5 級需要新實驗，超出工程範疇。 This chapter's scope: Tiers 1-3 are engineering problems and solvable here. Tiers 4-5 require new experiments, beyond engineering.
工具 1

Quarto — RMarkdown 的後繼者

Quarto 是 Posit (前 RStudio) 推出的下一代語言無關文件系統，支援 R / Python / Julia / Observable JS 在同一份 .qmd 檔案。一份 .qmd 可同時 render 為 HTML / PDF / docx / 投影片 / 整本書。對於需要嵌入分析的 supplementary 是首選。

Quarto is Posit's (formerly RStudio) language-agnostic next-gen document system: R / Python / Julia / Observable JS in one .qmd file. One source renders to HTML / PDF / docx / slides / a full book. Ideal for analysis-embedded supplementaries.

uv add scanpy pandas numpy pydeseq2 jupyterlab

uv lock

# Reproduce exactly (years later)
uv sync --frozen      # installs from uv.lock, fails if drift

# Run reproducibly
uv run python analysis.py

# In R project
install.packages("renv")
renv::init()         # detects packages from .R / .qmd files

# Lock current state
renv::snapshot()      # writes renv.lock with versions + sources (CRAN/Bioc/GitHub)

# Years later, on a fresh machine
renv::restore()       # installs exact versions from renv.lock

FAIR

FAIR 資料原則與存放

FAIR = Findable Accessible Interoperable Reusable。多數期刊 (Nature、Cell、PLoS、AAAS) 與資助機構 (NIH、ERC) 已強制要求論文程式碼與資料符合 FAIR。

FAIR = Findable Accessible Interoperable Reusable. Most journals (Nature, Cell, PLoS, AAAS) and funders (NIH, ERC) now require paper code/data to be FAIR-compliant.

📍 Findable

有唯一識別碼 (DOI、accession)，並登錄在搜尋引擎可找到的地方。實作：Zenodo 取得 DOI、GitHub repo 連結 ORCID、GEO/SRA/ENA/PRIDE 取得 accession。

Has a unique identifier (DOI, accession) registered in searchable indices. How: mint a Zenodo DOI, link the GitHub repo to ORCID, deposit to GEO/SRA/ENA/PRIDE for accessions.

🔓 Accessible

可由標準協定取得 (HTTPS、open API)，即使資料受管制也要說明取得程序 (dbGaP、EGA)。實作：避免「contact author for data」——這幾乎等於不可再現。

Retrievable via standard protocols (HTTPS, open APIs). Even if data is controlled (dbGaP, EGA), the access process must be documented. How: avoid "contact author for data" — that's effectively not reproducible.

🔗 Interoperable

使用社群標準格式 (FASTQ、BAM、VCF、AnnData、OME-Zarr、SBML)，搭配 ontology (Cell Ontology、Disease Ontology、HPO)。實作：metadata 用 OBO Foundry ontologies 標註。

Uses community standard formats (FASTQ, BAM, VCF, AnnData, OME-Zarr, SBML) and ontologies (Cell Ontology, Disease Ontology, HPO). How: annotate metadata with OBO Foundry ontologies.

♻️ Reusable

充足 metadata 與授權聲明 (CC-BY for data、MIT/Apache for code)，符合社群 metadata 標準 (MIAME、MINSEQE)。實作：每個 dataset 附 README 與 LICENSE。

Rich metadata + license (CC-BY for data, MIT/Apache for code), conforms to community metadata standards (MIAME, MINSEQE). How: attach a README + LICENSE per dataset.

💡
實務 checklist：論文發表前——(1) GitHub repo → Zenodo DOI；(2) raw data → GEO/SRA；(3) AnnData → CellxGene 或 Figshare；(4) container → DockerHub/GHCR 並寫進 README；(5) snakemake/nextflow workflow → snakemake-workflows / nf-core。 Practical checklist: Before submission — (1) GitHub repo → Zenodo DOI; (2) raw data → GEO/SRA; (3) AnnData → CellxGene or Figshare; (4) container → DockerHub/GHCR, document in README; (5) snakemake/nextflow workflow → snakemake-workflows / nf-core.

📝 自我檢測 (3 題)

Q1. 下列何者最不符合 FAIR data 原則？

Q1. Which is least FAIR-compliant?

A. 把 RNA-seq 上傳到 GEO 並取得 GSE accessionA. Uploading RNA-seq to GEO with a GSE accession
B. GitHub repo 連結 Zenodo DOI 並寫 LICENSEB. GitHub repo with a Zenodo DOI and LICENSE
C. 在論文 supplementary 寫「Code available from the corresponding author upon reasonable request」C. Supplementary note "Code available from the corresponding author upon reasonable request"
D. 用 OBO 標準 ontology 標註 cell typeD. Annotating cell types with OBO ontologies
答案 C。「contact the author」幾乎等於 not Accessible——研究發現絕大多數此類聲明在 5 年後無人回覆。應該存到 Zenodo / GitHub 等永久 archive。
Answer: C. "Contact the author" is effectively not Accessible — studies show most such statements get no reply after 5 years. Use permanent archives like Zenodo/GitHub.

Q2. 為什麼 pip freeze > requirements.txt 不足以再現環境？

Q2. Why is pip freeze > requirements.txt insufficient?

A. 它太冗長A. It's too verbose
B. 它沒有檔案 hash、沒有跨平台支援、不記錄 OS-level 二進位 (如 samtools)B. No hashes, no cross-platform spec, doesn't capture OS-level binaries (e.g. samtools)
C. 它只支援 Python 3.6 以下C. It only works for Python ≤3.6
D. 沒有問題D. There's no problem
答案 B。完整再現需要 hash (避免被替換)、跨平台 wheel、OS 工具版本。用 uv.lock / conda-lock 或 Dockerfile + image digest。
Answer: B. True reproducibility needs hashes (prevent swap), cross-platform wheels, and OS-tool versions. Use uv.lock / conda-lock or Dockerfile with image digest.

Q3. You want one .qmd file with R and Python code blocks, rendered to HTML + PDF — use what?

A. QuartoA. Quarto
