動機

CLI 何時不夠用？

大多數生醫資訊工具是 CLI (e.g. samtools、scanpy.cli)。CLI 簡單、易組合、適合 batch。但有四種情境 CLI 力不從心：

  • 互動探索：濕實驗同事要 click + filter + 重畫 UMAP，不會寫 Python，需要點按鈕的界面。
  • 跨組共享資料：三個 lab 想查同一個 variant database，每個 lab 自己跑 CLI 太重複，集中式 API 更合理。
  • 長時間運算 + 進度回報：跑 100 個樣本要 2 小時，使用者想看即時 progress bar。
  • 跟其他系統整合：EHR / LIMS / Galaxy 想呼叫你的 alignment service。

本章六個主題：FastAPI 基礎、plumber (R)、Dashboard 工具比較、REST vs GraphQL、WebSockets / SSE、部署。

Most bioinformatics tools are CLIs (e.g. samtools, scanpy). CLIs are simple, composable, batch-friendly. But four scenarios outgrow a CLI:

  • Interactive exploration: A wet-lab collaborator wants to click + filter + replot UMAP without writing Python.
  • Cross-lab data sharing: Three labs query the same variant DB — running CLIs locally duplicates effort; a central API is cleaner.
  • Long jobs with progress: A 100-sample run takes 2 hours; users want a live progress bar.
  • System integration: EHR / LIMS / Galaxy needs to call your alignment service.

The six topics: FastAPI basics, plumber (R), Dashboard tools compared, REST vs GraphQL, WebSockets / SSE, deployment.

💡
口訣：「CLI 給程式用，Web 給人用、給其他服務用。」做之前先問：誰是我的使用者？ Mantra: "CLI for code; Web for humans and other services." Before building: ask who your user is.

一、FastAPI

FastAPI 是 2025 年 Python 生態最熱門的 API 框架：基於 Pydantic 做 request/response 驗證，原生 async，自動生成 OpenAPI 文件 (即 Swagger UI) 在 /docs，支援 dependency injection 與 BackgroundTasks。

FastAPI is the dominant Python API framework in 2025: Pydantic-based request/response validation, native async, auto-generated OpenAPI docs at /docs, dependency injection, BackgroundTasks.

main.py — variant lookup API
from fastapi import FastAPI, HTTPException, Query
from pydantic import BaseModel, Field

app = FastAPI(title="variant-svc", version="0.1.0")

class Variant(BaseModel):
    chrom: str = Field(..., pattern=r"^chr[\dXYM]+$")
    pos:   int = Field(..., ge=1)
    ref:   str
    alt:   str
    qual:  float | None = None

class VariantList(BaseModel):
    gene: str
    n:    int
    variants: list[Variant]

@app.get("/variants/{gene}", response_model=VariantList)
async def list_variants(
    gene: str,
    chrom: str | None = Query(None, pattern=r"^chr[\dXYM]+$"),
    limit: int = Query(100, ge=1, le=1000),
):
    rows = await db.fetch_variants(gene, chrom, limit)
    if not rows:
        raise HTTPException(404, f"No variants for {gene}")
    return VariantList(gene=gene, n=len(rows), variants=rows)

deps.py — dependency injection
from fastapi import Depends, Header, HTTPException
import asyncpg

async def get_pool():
    if not hasattr(get_pool, "_p"):
        get_pool._p = await asyncpg.create_pool("postgres://...")
    return get_pool._p

async def require_api_key(x_api_key: str = Header(...)):
    if not await verify_key(x_api_key):
        raise HTTPException(401, "invalid API key")
    return x_api_key

@app.get("/admin/stats")
async def stats(pool = Depends(get_pool), _key = Depends(require_api_key)):
    async with pool.acquire() as conn:
        return {"n_variants": await conn.fetchval("SELECT count(*) FROM variants")}
background.py — long-running jobs
from fastapi import BackgroundTasks
from uuid import uuid4

jobs: dict[str, dict] = {}

def _align_worker(job_id: str, sample: str):
    jobs[job_id]["status"] = "running"
    # ... run minimap2/bwa ...
    jobs[job_id]["status"] = "done"
    jobs[job_id]["bam"]    = f"/data/{sample}.bam"

@app.post("/align")
async def align(sample: str, tasks: BackgroundTasks):
    job_id = str(uuid4())
    jobs[job_id] = {"status": "queued"}
    tasks.add_task(_align_worker, job_id, sample)
    return {"job_id": job_id}

@app.get("/align/{job_id}")
async def status(job_id: str):
    return jobs.get(job_id, {"error": "unknown job"})

# For heavy workloads, swap BackgroundTasks for Celery/Dramatiq/arq + Redis
flask.py — small Flask still has a place
from flask import Flask, jsonify, request

app = Flask(__name__)

@app.route("/gc", methods=["POST"])
def gc():
    seq = request.json.get("seq", "")
    gc = sum(1 for c in seq if c in "GCgc") / len(seq) if seq else 0
    return jsonify({"seq": seq, "gc": gc})

# Flask is fine for: tiny tools, demo apps, sync-only endpoints,
# or when team is already Flask-fluent. Use FastAPI for new projects.

二、R plumber 與 Shiny

R 對應的 API 框架是 plumber——用 #* 註解把任何 R 函式變 REST endpoint。對需要互動 dashboard 的場景，R 的 Shiny 是黃金標準 (UI + reactive server)。

R's API framework is plumber — #* annotations turn any R function into a REST endpoint. For interactive dashboards, R's Shiny is the gold standard (UI + reactive server).

plumber.R
library(plumber)

#* @apiTitle Variant Service (R)
#* @apiVersion 0.1.0

#* List variants in a gene
#* @param gene  Gene symbol
#* @param chrom Optional chromosome filter
#* @param limit Max rows (default 100)
#* @get /variants/<gene>
function(gene, chrom = NULL, limit = 100) {
  rows <- DBI::dbGetQuery(conn,
    "SELECT chrom,pos,ref,alt FROM variants WHERE gene=$1 LIMIT $2",
    params = list(gene, as.integer(limit)))
  list(gene = gene, n = nrow(rows), variants = rows)
}

#* Compute GC of a sequence
#* @param seq DNA sequence
#* @post /gc
function(seq) {
  bases <- strsplit(seq, "")[[1]]
  list(seq = seq, gc = mean(bases %in% c("G","C","g","c")))
}

# Run: plumber::pr("plumber.R") |> plumber::pr_run(port = 8000)
# Swagger UI at http://localhost:8000/__docs__/
app.R — minimal Shiny QC dashboard
library(shiny)
library(ggplot2)
library(scqcR)

ui <- fluidPage(
  titlePanel("scRNA-seq QC Explorer"),
  sidebarLayout(
    sidebarPanel(
      fileInput("file", "Upload .h5ad / .rds"),
      sliderInput("min_counts", "Min counts/cell",
                  0, 5000, 500, step = 100),
      sliderInput("max_mt", "Max MT %", 0, 100, 10)
    ),
    mainPanel(
      plotOutput("qcPlot"),
      verbatimTextOutput("stats")
    )
  )
)

server <- function(input, output) {
  sce <- reactive({ req(input$file); read_sce(input$file$datapath) })
  filtered <- reactive({
    filter_cells(sce(), min_counts = input$min_counts, max_mt = input$max_mt)
  })
  output$qcPlot <- renderPlot({
    ggplot(as.data.frame(colData(filtered())),
           aes(total_counts, pct_mt)) + geom_point(alpha = .4)
  })
  output$stats <- renderPrint({
    sprintf("%d cells pass / %d total", ncol(filtered()), ncol(sce()))
  })
}

shinyApp(ui, server)

import streamlit as st
import scanpy as sc, matplotlib.pyplot as plt

st.title("scRNA-seq QC Explorer")
f = st.file_uploader("Upload .h5ad", type=["h5ad"])
if f:
    adata = sc.read_h5ad(f)
    mc = st.sidebar.slider("Min counts/cell", 0, 5000, 500)
    mm = st.sidebar.slider("Max MT %", 0, 100, 10)
    keep = (adata.obs.total_counts >= mc) & (adata.obs.pct_counts_mt <= mm)
    st.write(f"**{keep.sum()}** cells pass / {adata.n_obs} total")
    fig, ax = plt.subplots()
    ax.scatter(adata.obs.total_counts[keep],
               adata.obs.pct_counts_mt[keep], alpha=0.4, s=4)
    st.pyplot(fig)

sse.py — server-sent events for progress
from fastapi.responses import StreamingResponse
import asyncio, json

@app.get("/align/{job_id}/stream")
async def stream(job_id: str):
    async def events():
        while jobs[job_id]["status"] != "done":
            payload = json.dumps({
                "status": jobs[job_id]["status"],
                "progress": jobs[job_id].get("pct", 0)
            })
            yield f"data: {payload}\n\n"
            await asyncio.sleep(1)
        yield f"data: {{\"status\":\"done\"}}\n\n"
    return StreamingResponse(events(), media_type="text/event-stream")

# Browser: const es = new EventSource('/align/abc/stream');
#          es.onmessage = e => updateProgressBar(JSON.parse(e.data));
websocket.py — bi-directional
from fastapi import WebSocket

@app.websocket("/cluster/live")
async def cluster_ws(ws: WebSocket):
    await ws.accept()
    try:
        while True:
            msg = await ws.receive_json()
            if msg["action"] == "recluster":
                result = await recluster(msg["resolution"])
                await ws.send_json({"clusters": result})
    except Exception:
        await ws.close()

# Use when both sides need to push updates (e.g. multi-user dashboard).

Shiny: shinyapps.io (free) / Posit Connect (enterprise) / shinylive (runs entirely in-browser via WebR, no server).   --bind 0.0.0.0:8000 \

server {
    listen 443 ssl http2;
    server_name variants.lab.org;
    ssl_certificate     /etc/letsencrypt/live/variants.lab.org/fullchain.pem;

        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_http_version 1.1;

Q1. FastAPI 相對於 Flask 的主要優點？


Q1. Main advantage of FastAPI over Flask?




A. Flask 不能寫 REST APIA. Flask can't build REST APIs


B. FastAPI 用 SQL 比較快B. FastAPI runs SQL faster


C. Pydantic 自動驗證 + 自動 OpenAPI 文件 + 原生 asyncC. Pydantic validation + auto OpenAPI docs + native async


D. FastAPI 不需要 uvicornD. FastAPI doesn't need uvicorn




答案 C。Flask 仍然好用（特別是小工具、團隊已熟悉），但新專案建議 FastAPI 是因為三件事：Pydantic 自動驗證 request/response、Swagger 文件免費送、async/await 支援高並發 I/O。


Answer: C. Flask remains fine for small tools or teams already fluent in it; new projects favour FastAPI for three reasons: Pydantic auto-validates payloads, Swagger docs come free, and native async handles high-concurrency I/O.







Q2. 想給 wet-lab 同事一個能 click + filter + 拉 slider 的 R 介面，最直接的選擇？


Q2. Most direct choice for a click + filter + slider UI for wet-lab colleagues in R?




A. 用 plumber 開 REST APIA. Build a REST API with plumber


B. Shiny (reactive UI + server)B. Shiny (reactive UI + server)


C. 寫一個 CLI 加 promptC. Write a CLI with prompts


D. 把分析寫成 Rmarkdown 寄給他們D. Email them an Rmarkdown




答案 B。Shiny 是 R 生態的標準 dashboard 工具，reactive 模型讓 input 改變自動觸發 plot 重畫；plumber 是給「程式呼叫」的 REST API；CLI/Rmd 沒有互動 UI。


Answer: B. Shiny is R's canonical dashboard tool; its reactive model auto-redraws plots when inputs change. plumber is a REST API for other programs to call. CLI/Rmd lack interactive UI.







Q3. 100 樣本的 alignment job 跑 2 小時，前端想看即時進度。最簡單合適的協定？


Q3. 100-sample alignment job runs 2 hours; UI needs live progress. Simplest fitting protocol?




A. Server-Sent Events (SSE) — 單向 stream，HTTP 友善A. Server-Sent Events (SSE) — one-way stream, HTTP-friendly


B. 每 5 秒 GET 一次 status (polling)B. Poll status every 5s


C. 用 GraphQL subscriptionC. GraphQL subscription


D. 用 email 通知D. Email when done




答案 A。SSE 是 server → client 的單向 stream，跑在普通 HTTP 上 (穿透 firewall/反向 proxy 容易)，瀏覽器內建 EventSource API。WebSocket 也可以但通常是「需要雙向」才用。Polling 浪費資源；email 不算即時。


Answer: A. SSE is a one-way server-to-client stream over plain HTTP (passes firewalls/proxies easily) with built-in browser EventSource. WebSocket also works but is overkill for one-way; polling wastes resources; email isn't real-time.
