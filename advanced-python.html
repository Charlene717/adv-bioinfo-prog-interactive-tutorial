進階 Python：把腳本升級為函式庫

Type hints、dataclasses、generator、decorator、context manager、async/await——讓你的 scRNA-seq / variant calling 程式從「能跑」變成「可被其他人安全引用」。

Type hints, dataclasses, generators, decorators, context managers, async/await — turn your scRNA-seq / variant-calling scripts from "it runs" into "others can safely import it."

動機

為什麼研究腳本需要「進階化」？

大多數生醫資訊腳本一開始是 Jupyter notebook 中的 for cell in cells: ...。但當你的腳本被同事 import、被審稿者要求重現、被引用到論文方法章節，就會暴露三類問題：

  • 型別模糊：函式參數 data 到底是 pd.DataFrame、AnnData 還是 np.ndarray？呼叫者只能看 source code。
  • 狀態管理混亂：檔案開了忘關、temp file 沒清、try/except 漏寫——pipeline 失敗時留下半成品檔案。
  • 難以擴充：要從處理 1 個樣本變成處理 1000 個，發現迴圈寫死、記憶體炸掉、不能 streaming。

本章六個主題：type hints、dataclasses、generators、decorators、context managers、async/await，是解決上述問題的核心工具。

Most bioinformatics scripts start as a for cell in cells: ... in a Jupyter notebook. But once your script gets imported by a colleague, reviewed for reproducibility, or cited in a methods section, three pain points emerge:

  • Ambiguous types: Is the parameter data a pd.DataFrame, an AnnData, or an np.ndarray? Callers must read source code to know.
  • State management chaos: Files opened without closing, temp files leaked, missing try/except — pipelines fail and leave half-written outputs.
  • Hard to scale: Going from 1 sample to 1000 reveals hard-coded loops, OOM crashes, no streaming.

The six topics here — type hints, dataclasses, generators, decorators, context managers, async/await — are the core tools that solve these problems.

💡
學習策略：不要試圖一次學完。把每個工具當成「某個常見痛點」的解藥——當你下次寫到那個痛點，就用上對應工具。本頁附「何時應該用」的決策樹。 Learning strategy: Don't try to learn all at once. Treat each tool as a cure for a specific pain point — next time you hit that pain, reach for the tool. A decision tree on "when to use what" is included below.
主題 1

一、Type Hints 與 mypy / pyright

Type hints 是 Python 3.5+ 引入的 選擇性靜態型別標註。CPython 不會在 runtime 強制檢查，但工具（mypy / pyright）會在 CI 時抓出型別錯誤。對於 Bioconductor 等級的套件，type hints 是文件、IDE 自動完成、與重構安全網的合一解方。

Type hints are optional static type annotations introduced in Python 3.5+. CPython doesn't enforce them at runtime, but tools (mypy/pyright) catch type errors in CI. For Bioconductor-grade libraries, type hints serve as documentation, IDE autocomplete, and refactor-safety all in one.

basic.py
# Python 3.10+ syntax (no need for `from typing import Optional`)
from pathlib import Path
import pandas as pd

def read_counts(
    path: Path | str,
    sep: str = "\t",
    min_cells: int = 3,
) -> pd.DataFrame:
    """Read gene-count matrix, drop genes seen in < min_cells cells."""
    df = pd.read_csv(path, sep=sep, index_col=0)
    mask = (df > 0).sum(axis=1) >= min_cells
    return df.loc[mask]

# Callers + IDEs both know: path can be Path or str, return is DataFrame
counts = read_counts("data/sample.tsv", min_cells=5)
generics.py
from typing import TypeVar, Generic, Sequence
import numpy as np

T = TypeVar("T", bound=np.number)  # T must be numeric

def zscore(x: np.ndarray[tuple[int, ...], np.dtype[T]]) -> np.ndarray:
    return (x - x.mean(axis=0)) / x.std(axis=0)

# Generic container
class GeneSet(Generic[T]):
    def __init__(self, genes: Sequence[T]) -> None:
        self.genes: list[T] = list(genes)
    def first(self) -> T:
        return self.genes[0]
protocol.py — structural typing
from typing import Protocol, runtime_checkable

@runtime_checkable
class Normalizer(Protocol):
    def fit_transform(self, X: np.ndarray) -> np.ndarray: ...

# Anything that has `fit_transform` matches — no inheritance needed
def run_pipeline(X: np.ndarray, norm: Normalizer) -> np.ndarray:
    return norm.fit_transform(X)

# sklearn's StandardScaler, your own LogNormalize — both satisfy Normalizer

二、Dataclasses 與 Pydantic

當你的函式參數變多 (min_cells, max_mt, sample_id, batch, ...)，把它們包進一個物件比每次傳 8 個參數更乾淨。三種選擇：

When function signatures grow (min_cells, max_mt, sample_id, batch, ...), wrapping them in an object is cleaner than passing 8 arguments. Three choices:

工具 標準函式庫? Runtime 型別檢查 JSON / 序列化 適用場景
@dataclass✅❌手動內部結構、配置物件、不需驗證
attrs❌可選 (validators)搭配 cattrs比 dataclass 更多功能
Pydantic v2❌✅原生 (Rust 核心)API 邊界、設定檔、外部資料
dataclass.py
from dataclasses import dataclass, field
from pathlib import Path

@dataclass(frozen=True)  # immutable → hashable, safe as dict key
class QCParams:
    min_features: int = 200
    max_features: int = 6000
    max_mt_pct: float = 10.0
    species: str = "human"
    excluded_chrs: tuple[str, ...] = field(default_factory=lambda: ("chrY", "chrM"))

def run_qc(adata, params: QCParams):
    keep = (adata.obs.nFeature >= params.min_features) & \
           (adata.obs.nFeature <= params.max_features) & \
           (adata.obs.pct_mt   <= params.max_mt_pct)
    return adata[keep].copy()
pydantic.py — for YAML configs / API bodies
from pydantic import BaseModel, Field, field_validator
import yaml

class QCConfig(BaseModel):
    min_features: int = Field(200, ge=0, le=10000)
    max_features: int = Field(6000, ge=100)
    max_mt_pct:   float = Field(10.0, ge=0, le=100)
    species:      str   = "human"

    @field_validator("species")
    @classmethod
    def _known_sp(cls, v: str) -> str:
        if v not in {"human", "mouse", "rat"}:
            raise ValueError(f"species must be human/mouse/rat, got {v}")
        return v

# Auto-validate when loading from YAML
with open("config.yaml") as f:
    cfg = QCConfig(**yaml.safe_load(f))

主題 3

三、Generators 與 itertools — 處理巨型檔案

Generator 用 yield 一次只產出一個元素，不會把整個檔案讀進記憶體。對於 FASTQ (數十 GB)、VCF (數百萬列)、AnnData (數百萬細胞) 的處理至關重要。

A generator uses yield to emit one item at a time without loading the whole file into memory. Critical for FASTQ (tens of GB), VCF (millions of lines), AnnData (millions of cells).

記憶體比較模擬器

調整檔案大小，比較「整檔讀入」 vs 「generator 串流」的記憶體佔用。

Adjust file size; compare peak memory of "read-all" vs "generator streaming."

stream_fastq.py
from typing import Iterator
import gzip

def iter_fastq(path: str) -> Iterator[tuple[str, str, str]]:
    """Yield (header, sequence, quality) — never loads whole file."""
    opener = gzip.open if path.endswith(".gz") else open
    with opener(path, "rt") as f:
        while True:
            h = f.readline().rstrip()
            if not h: return
            seq = f.readline().rstrip()
            f.readline()           # skip '+'
            qual = f.readline().rstrip()
            yield h, seq, qual

# Even on a 100GB FASTQ, this uses ~10MB RAM
gc_counts = 0
for hdr, seq, qual in iter_fastq("reads.fq.gz"):
    gc_counts += seq.count("G") + seq.count("C")
itertools.py
from itertools import islice, groupby, chain

# Take first 1000 reads without loading rest
first_1k = list(islice(iter_fastq("reads.fq.gz"), 1000))

# Group VCF lines by chromosome (file MUST be sorted by chrom)
def iter_vcf(path: str):
    with open(path) as f:
        for line in f:
            if line.startswith("#"): continue
            yield line.split("\t")

for chrom, recs in groupby(iter_vcf("variants.vcf"), key=lambda r: r[0]):
    n = sum(1 for _ in recs)
    print(f"{chrom}: {n} variants")
pipeline.py — composable generator pipeline
def trim_low_qual(reads, min_q: int = 20):
    for hdr, seq, qual in reads:
        avg = sum(ord(c)-33 for c in qual) / len(qual)
        if avg >= min_q:
            yield hdr, seq, qual

def to_fasta(reads):
    for hdr, seq, _ in reads:
        yield f">{hdr[1:]}\n{seq}\n"

# Lazy pipeline: nothing runs until you iterate
pipe = to_fasta(trim_low_qual(iter_fastq("reads.fq.gz"), min_q=25))
with open("clean.fasta", "w") as out:
    out.writelines(pipe)
主題 4

四、Decorators — 切片式行為注入

Decorator 是「接受函式並回傳新函式」的高階函式，最常見於 cache、計時、log、retry、權限驗證等「跨切面關注點」。FastAPI 的路由、joblib 的 cache、jit、numba.jit 全都是 decorator。

A decorator is a higher-order function that takes a function and returns a new one — typical uses include caching, timing, logging, retries, authorization. FastAPI routes, joblib cache, JIT, numba.jit are all decorators.

timing.py — measure runtime + memory
import functools, time, tracemalloc, logging
from typing import Callable, ParamSpec, TypeVar

P = ParamSpec("P"); R = TypeVar("R")
log = logging.getLogger(__name__)

def timed(fn: Callable[P, R]) -> Callable[P, R]:
    @functools.wraps(fn)
    def wrapper(*a: P.args, **kw: P.kwargs) -> R:
        tracemalloc.start()
        t0 = time.perf_counter()
        try:
            return fn(*a, **kw)
        finally:
            dt = time.perf_counter() - t0
            cur, peak = tracemalloc.get_traced_memory()
            tracemalloc.stop()
            log.info(f"{fn.__name__}: {dt:.2f}s, peak {peak/1e6:.1f} MB")
    return wrapper

@timed
def run_clustering(adata, resolution: float = 0.5):
    ...
cache.py — memoization + disk cache
from functools import lru_cache
from joblib import Memory

# In-memory LRU (per-process)
@lru_cache(maxsize=128)
def fetch_gene_info(symbol: str) -> dict:
    # Slow REST call to Ensembl/NCBI
    return requests.get(f"https://rest.ensembl.org/lookup/symbol/human/{symbol}").json()

# Persistent disk cache (joblib) — survives process restart
mem = Memory("./.cache", verbose=0)

@mem.cache
def compute_pca(X, n_comp: int = 50):
    from sklearn.decomposition import PCA

retry.py — for flaky REST calls
from tenacity import retry, stop_after_attempt, wait_exponential, retry_if_exception_type
import requests

@retry(
    stop=stop_after_attempt(5),
    wait=wait_exponential(multiplier=1, min=2, max=30),
    retry=retry_if_exception_type(requests.HTTPError),
)
def query_ensembl(endpoint: str) -> dict:
    r = requests.get(f"https://rest.ensembl.org{endpoint}", timeout=10)
    r.raise_for_status()
    return r.json()
主題 5

五、Context Managers — 確保資源被釋放

with 區塊在進入時取得資源、在離開時 (即使是例外) 必定釋放。Bioinformatics 中常見用途：開啟 BAM/VCF 檔案、申請 GPU 記憶體、切換工作目錄、暫時調高 logging level、locking。

A with block acquires a resource on entry and always releases on exit (even on exception). Common bioinformatics uses: opening BAM/VCF files, GPU memory, changing working directory, raising logging level, locking.

contextlib.py — quick @contextmanager
from contextlib import contextmanager
import os, tempfile, shutil

@contextmanager
def cd(path):
    prev = os.getcwd()
    os.chdir(path)
    try:
        yield path
    finally:
        os.chdir(prev)        # always restored — even on exception

@contextmanager
def tempdir():
    d = tempfile.mkdtemp()
    try:
        yield d
    finally:
        shutil.rmtree(d, ignore_errors=True)

with tempdir() as d, cd(d):
    os.system("samtools sort input.bam -o sorted.bam")
    # tempdir is deleted automatically here
class_based.py — for stateful resources
import pysam, logging

class BamReader:
    def __init__(self, path: str):
        self.path = path
    def __enter__(self):
        self.bam = pysam.AlignmentFile(self.path, "rb")
        return self.bam
    def __exit__(self, exc_type, exc_val, tb):
        self.bam.close()
        if exc_type:
            logging.error(f"BAM read failed on {self.path}: {exc_val}")
        return False   # don't suppress exception

with BamReader("sample.bam") as bam:
    for read in bam.fetch("chr1", 1_000_000, 1_001_000):
        ...
multi.py — ExitStack for variable N resources
from contextlib import ExitStack

def merge_samples(bam_paths: list[str], out: str):
    with ExitStack() as stack:
        bams = [stack.enter_context(pysam.AlignmentFile(p, "rb")) for p in bam_paths]
        writer = stack.enter_context(pysam.AlignmentFile(out, "wb", template=bams[0]))
        for bam in bams:
            for read in bam:
                writer.write(read)
        # All N files closed automatically on exit
主題 6

六、async / await — 並行 I/O

當你要呼叫 100 個 Ensembl REST endpoint，序列化執行需要 100 × 200ms = 20 秒；async 並行可以縮到 ~300ms。async 適合 I/O 密集 (網路、檔案)，不適合 CPU 密集 (CPU 重的工作用 multiprocessing — 見 Block B)。

Querying 100 Ensembl REST endpoints serially: 100 × 200ms = 20s. Async concurrently: ~300ms. Async is for I/O-bound work (network, disk), not CPU-bound (use multiprocessing for CPU — see Block B).

async_rest.py — 100 Ensembl queries in parallel
import asyncio, httpx

async def fetch_gene(client: httpx.AsyncClient, symbol: str) -> dict:
    r = await client.get(f"/lookup/symbol/human/{symbol}", timeout=10)
    r.raise_for_status()
    return r.json()

async def main(symbols: list[str]) -> list[dict]:
    async with httpx.AsyncClient(
        base_url="https://rest.ensembl.org",
        headers={"Accept":"application/json"},
        limits=httpx.Limits(max_connections=20),
    ) as client:
        return await asyncio.gather(*(fetch_gene(client, s) for s in symbols))

genes = asyncio.run(main(["BRCA1", "TP53", "EGFR"] * 30))
async_db.py — async PostgreSQL with asyncpg
import asyncio, asyncpg

async def load_variants(sample_ids: list[str]):
    pool = await asyncpg.create_pool("postgresql://user@host/genomics")
    async with pool.acquire() as conn:
        rows = await conn.fetch(
            "SELECT chrom, pos, ref, alt FROM variants WHERE sample_id = ANY($1)",
            sample_ids
        )
    await pool.close()
    return rows
🚫
常見錯誤：在 async 函式中呼叫 requests.get() 或 time.sleep() 會 block 整個 event loop——讓並行失效。請改用 httpx.AsyncClient、asyncio.sleep()。 Common mistake: Calling requests.get() or time.sleep() inside an async function blocks the entire event loop — defeating concurrency. Use httpx.AsyncClient, asyncio.sleep() instead.
決策樹

該用哪個工具？

情境 → 工具對照

Q1→「我希望 IDE 跳出自動完成 + CI 抓型別錯誤」 → type hints + mypy/pyright"I want IDE autocomplete + CI to catch type errors" → type hints + mypy/pyright
Q2→「我有一堆參數想包起來」 → @dataclass (內部) 或 BaseModel (來自 YAML/API)"I want to bundle parameters" → @dataclass (internal) or BaseModel (from YAML/API)
Q3→「我的 FASTQ / VCF 太大讀不進記憶體」 → generator + itertools"My FASTQ/VCF doesn't fit in RAM" → generator + itertools
Q4→「我想對許多函式加上同樣的計時 / cache / retry」 → decorator"I want to add timing/cache/retry to many functions" → decorator
Q5→「我要確保臨時檔/連線一定被釋放」 → context manager"I must guarantee cleanup of temp files/connections" → context manager
Q6→「我要打 100 個 REST API」 → async / await (I/O bound)；「我要對 100 樣本做重 CPU 計算」 → multiprocessing (B3)"I'm calling 100 REST APIs" → async/await; "100 samples, heavy CPU" → multiprocessing (B3)

Q1. 下列何者最適合用 generator 而不是 list comprehension？

Q1. Which is best suited for a generator rather than a list comprehension?

A. 把 1000 個整數 +1 並馬上加總A. Add 1 to 1000 ints and sum immediately
B. 已知資料量小且需要重複迭代B. Small data that needs repeated iteration
C. 串流讀取 100GB FASTQ，每次只處理一筆 readC. Streaming a 100GB FASTQ, one read at a time
D. 將資料轉成 NumPy array 並一次給 sklearnD. Convert to NumPy array, feed to sklearn at once
答案 C。Generator 適合資料量大、單次線性掃描、不需重複迭代的情境，典型用途就是 FASTQ/VCF 串流。
Answer: C. Generators shine when data is large, scanned once linearly, with no repeated iteration — FASTQ/VCF streaming is the canonical case.

Q2. 把 requests.get() 包進 async def 並用 asyncio.gather() 並行呼叫 100 次——預期會發生什麼？

Q2. Wrapping requests.get() in async def and calling 100× with asyncio.gather() — what happens?

A. 100 個請求真正並行，速度提升 100 倍A. 100 requests truly run in parallel, 100× speedup
B. 請求仍序列化執行，因為 requests 是 blocking I/O，會 block event loopB. Still sequential — requests is blocking I/O and blocks the event loop
C. 速度提升取決於 CPU 核心數C. Speedup scales with CPU cores
D. Python 會自動轉成 async 版本D. Python auto-converts to async
答案 B。async/await 需要對應的 async-aware client (httpx.AsyncClient、aiohttp)。blocking I/O 函式會 block event loop 讓 gather 退化成序列化。
Answer: B. async/await needs async-aware clients (httpx.AsyncClient, aiohttp). Blocking I/O functions block the event loop and reduce gather to sequential.

Q3. 下列關於 @dataclass(frozen=True) 的描述何者正確？(多選)

Q3. Which statements about @dataclass(frozen=True) are correct? (multi)

A. 物件不可修改 (immutable)，可作為 dict 的 keyA. Object is immutable, usable as a dict key
B. 自動進行 runtime 型別檢查B. Automatically validates types at runtime
C. 可以從 YAML 自動載入並驗證C. Auto-loads & validates from YAML
D. 比一般 dataclass 節省記憶體 (僅 frozen 屬性，與 slots 是不同概念)D. Saves RAM (frozen and slots are different things)
答案 A。frozen 讓物件 immutable，因此可雜湊。B/C 是 Pydantic 的功能。記憶體節省屬於 slots=True，與 frozen 是兩件事。
